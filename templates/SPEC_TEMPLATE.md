# Dafny Spec Guide: [Module Name]

This document is a reference for the agent writing `logic/<module>.dfy`.
It is not executable — it describes the mapping from PRD sections to Dafny constructs.

---

## Mapping: PRD → Dafny

| PRD Section | Dafny Construct | Notes |
|---|---|---|
| §3 State Invariants | `invariant` inside `class` or `datatype` | Must hold after every method call |
| §4 Preconditions | `requires` clause on `method` | Caller is responsible for satisfying these |
| §4 Postconditions | `ensures` clause on `method` | Verifier proves these hold |
| §4 Failure modes | `returns (result: Result<T, Error>)` | Model errors as explicit return values |
| §5 Edge Cases | Separate `lemma` or additional `requires` | Prove each edge case is handled correctly |

## Dafny File Structure

```dafny
// logic/<module-name>.dfy
// Generated by: clarify-prove-code /prove
// Source: docs/PRD.md
// Verified: [dafny verify output — PASS / FAIL]

// ─────────────────────────────────────────────
// Data types
// ─────────────────────────────────────────────

datatype Error =
  | InvalidInput
  | InsufficientFunds
  | OutOfBounds
  // Add error variants from PRD §4 failure modes

// ─────────────────────────────────────────────
// State
// ─────────────────────────────────────────────

// Model the state of the module.
// Use `ghost` variables for logical state not visible at runtime.
class ModuleState {
  var field1: int
  var field2: seq<int>

  // State invariants from PRD §3
  predicate Valid()
    reads this
  {
    field1 >= 0 &&
    |field2| <= 1000
    // ... one clause per INV-N
  }
}

// ─────────────────────────────────────────────
// Operations
// ─────────────────────────────────────────────

// One method per operation in PRD §4
method OperationName(
  state: ModuleState,
  param1: int,
  param2: int
) returns (result: Result<int, Error>)
  // Preconditions from PRD §4.N
  requires param1 > 0
  requires state.Valid()
  // Postconditions from PRD §4.N
  ensures result.Ok? ==> state.Valid()
  ensures result.Ok? ==> result.value == param1 + param2
  // Failure mode: explicit
  ensures param1 <= 0 ==> result == Err(Error.InvalidInput)
{
  if param1 <= 0 {
    return Err(Error.InvalidInput);
  }
  // ... implementation
  return Ok(param1 + param2);
}

// ─────────────────────────────────────────────
// Edge case lemmas (from PRD §5)
// ─────────────────────────────────────────────

lemma EdgeCase_SameSource(state: ModuleState, amount: int)
  requires state.Valid()
  requires amount > 0
  ensures // prove the edge case is handled
{
  // proof body
}
```

## Verification Command

```sh
dafny verify logic/<module-name>.dfy
```

Expected output when all proofs pass:
```
Dafny program verifier finished with N verified, 0 errors
```

Do not proceed to `/code` until this output is confirmed.

## Common Pitfalls

- **Forgetting `reads`/`modifies` clauses.** Dafny requires explicit frame conditions on methods that read or modify heap state.
- **Off-by-one in sequences.** Use `0 <= i < |seq|` for index bounds in `ensures` clauses.
- **Circular invariants.** If two invariants depend on each other, Dafny may time out. Break them into separate predicates.
- **Missing base cases in lemmas.** Always prove the empty/zero case first.
